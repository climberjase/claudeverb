  <action>
    Create claudeverb/streamlit_app.py as a single-file Streamlit application:

    **Page config:** `st.set_page_config(page_title="ClaudeVerb", layout="wide")`

    **Sidebar (st.sidebar):**
    1. Title: "ClaudeVerb"
    2. Audio source: `st.radio("Audio Source", ["Bundled Sample", "Upload File"])`.
       - Bundled: `st.selectbox("Sample", list_samples())` then `get_sample(name)`.
       - Upload: `st.file_uploader("Upload Audio", type=["wav", "aiff", "flac", "ogg"])`. Load via `librosa.load(uploaded_file, sr=SAMPLE_RATE, mono=False)`, convert to float32. Call `.seek(0)` on uploaded_file before passing to librosa (Streamlit UploadedFile quirk).
    3. Algorithm: `st.selectbox("Algorithm", list(ALGORITHM_REGISTRY.keys()))`.
    4. Auto-generated controls from param_specs:
       - For each spec with type="knob": `st.slider(label, min, max, default, key=f"knob_{name}")`.
       - SKIP the "mix" knob (engine forces mix=100; UI wet/dry replaces it).
       - For each spec with type="switch": `st.select_slider(label, options=positions, value=default, format_func=lambda x, labels=spec["labels"], positions=spec["positions"]: labels[positions.index(x)], key=f"switch_{name}")`.
    5. Wet/Dry slider: `st.slider("Wet/Dry Mix", 0, 100, 75, key="wet_dry")`.
    6. Process button: `st.button("Process", type="primary", use_container_width=True)`.
    7. Reset button: `st.button("Reset to Defaults")` -- clears session_state algorithm params (use st.rerun()).

    **Session state initialization:**
    ```python
    for key in ["dry_audio", "wet_audio", "results", "audio_loaded"]:
        if key not in st.session_state:
            st.session_state[key] = None
    ```

    **Process button handler:**
    When clicked and audio is loaded:
    - Collect params from sidebar widget values (read from st.session_state using widget keys)
    - Call `engine.process_audio(algo_name, params, audio)`
    - Store results in session_state: dry_audio, wet_audio, results
    - Show `st.spinner("Processing...")` during execution

    **Main area (after processing):**
    If session_state.results is not None:

    1. **Audio Players** -- two columns side by side:
       - Left: "Input" with `st.audio(dry_audio, sample_rate=SAMPLE_RATE, loop=True)`
       - Right: "Output" with blended audio from `blend_wet_dry(dry, wet, wet_dry_value)` played via `st.audio(blended, sample_rate=SAMPLE_RATE, loop=True)`
       - Include `st.download_button` for processed audio (use soundfile to write WAV bytes to BytesIO)

    2. **Waveform** -- `st.pyplot(plot_waveform_comparison(dry, wet))`, close figure after

    3. **Spectrograms** -- mel comparison figure from results["figures"]["mel"] via `st.pyplot()`, close after

    4. **FFT Overlay** -- fft comparison figure from results["figures"]["fft"] via `st.pyplot()`, close after

    5. **Metrics** -- display using `st.metric()` in 4 columns:
       - Column layout: [RT60 broadband, DRR] [RT60 low, C80] [RT60 mid, C50] [RT60 high, Centroid Delta]

    6. **Impulse Response** -- show IR waveform via `st.audio(ir, sample_rate=SAMPLE_RATE)` and a small waveform plot

    Close all matplotlib figures after st.pyplot() calls to prevent memory leaks: `plt.close(fig)`.

    All visualizations stacked vertically (not tabbed) per locked decision. Use `st.subheader()` to label each section.

    **Important details:**
    - st.audio with numpy arrays: pass `sample_rate=SAMPLE_RATE` parameter
    - For stereo audio in st.audio: our (2, N) convention should work directly; if issues, transpose to (N, 2)
    - WAV download: use `io.BytesIO` + `soundfile.write(buf, data.T if stereo else data, SAMPLE_RATE, format="WAV")` then `st.download_button("Download WAV", buf.getvalue(), "processed.wav", "audio/wav")`
