  <action>
Create `claudeverb/algorithms/early_reflections.py` following the composable FDNCore pattern:

1. Start with C struct typedef comment at top of file (C portability convention).

2. Implement `EarlyReflections` class per the research design:
   - Constructor takes: `tap_delays: list[int]`, `tap_gains: list[float]`, `tap_lpf_coeffs: list[float]`, `diffuser_delays: list[int]`, `diffuser_feedback: float = 0.5`, `sample_rate: int = 48000`
   - Single `DelayLine` with max_delay = max(tap_delays) + 2
   - Per-tap `OnePole` filters using tap_lpf_coeffs
   - Cascade of `AllpassFilter` diffusers
   - Store base tap delays for size scaling

3. `process_sample(x: float) -> tuple[float, float]`:
   - Write input to delay line
   - Read each tap, apply per-tap OnePole, apply gain
   - Alternating L/R panning: even index taps -> right, odd -> left (per CONTEXT.md decision)
   - Sum L+R for mono diffusion input
   - Run through allpass diffuser cascade
   - Add diffused signal to both channels
   - Normalize output by `1.0 / sqrt(num_taps)` to prevent ER->FDN gain staging issues (Pitfall 1 from research)

4. `set_size_scale(scale: float)`:
   - Recalculate tap delays as `int(base_delay * scale)` clamped to valid range
   - Recreate delay line if new max exceeds current capacity

5. `reset()`: Reset delay line, all tap filters, all diffusers.

Create `tests/test_early_reflections.py` with the behavior tests listed above. Use Small Room config values from research for test fixtures.
