  <name>Task 1: Implement FDNReverb wrapper with param_specs, stereo output, freeze mode, and C export</name>
  <files>claudeverb/algorithms/fdn_reverb.py</files>
  <action>
    Add FDNReverb class to the existing fdn_reverb.py file (which already contains FDNCore from Plan 01).

    **FDNReverb(ReverbAlgorithm) class:**

    **__init__(self):**
    - Create self._fdn_core = FDNCore(BASE_DELAYS)
    - Create 3 input diffusers: AllpassFilter instances with prime delays [149, 233, 347] and gain=0.5
    - Create pre-delay line: DelayLine(max_delay=int(0.1 * 48000)) -- up to 100ms pre-delay
    - Initialize parameter state variables for all 6 knobs at defaults
    - Call self._initialize()

    **_initialize(self):**
    - Set default parameter values:
      - decay_time=50 (maps to ~2.5s RT60)
      - size=50 (maps to 1.0x scale)
      - damping=40 (moderate HF rolloff)
      - modulation=30 (mild chorus)
      - mix=75
      - pre_delay=0
      - switch1=0 (Normal)
      - switch2=0 (Stereo)
    - Apply defaults via update_params()

    **_process_impl(self, audio: np.ndarray) -> np.ndarray:**
    - Handle mono (N,) or stereo (2, N) input -> sum to mono if stereo
    - Create output array: shape (2, N), dtype float32
    - Sample-by-sample loop:
      1. Read from pre-delay line, write current sample
      2. Pass through 3 cascaded input diffusers (process_sample each)
      3. Distribute to 4 FDN channels: [diffused, diffused, diffused, diffused] (equal distribution per research recommendation)
      4. FDNCore.process_sample() -> get 4 channel outputs
      5. Stereo tap: L = out[0] + out[2], R = out[1] + out[3]
      6. Width crossmix based on stereo mode:
         - switch2=-1 (Mono): width=0.0 -> L=R=(L+R)/2
         - switch2=0 (Stereo): width=1.0
         - switch2=1 (Wide): width=2.0 (exaggerated stereo)
         - wet1 = (1 + width) / 2, wet2 = (1 - width) / 2
         - final_l = L * wet1 + R * wet2, final_r = R * wet1 + L * wet2
      7. Hard clip to [-1.0, 1.0]
      8. Store in output array
    - Return output (always stereo (2, N))

    **update_params(self, params: dict):**
    - Map each knob 0-100 to internal values:
      - decay_time: 0-100 -> 0.2s to 20.0s RT60 (exponential mapping: 0.2 * (100^(val/100)) gives good spread)
      - size: 0-100 -> 0.5x to 2.0x delay scale
      - damping: 0-100 -> 0.0 to 0.95 OnePole coefficient (0=bright, 100=dark)
      - modulation: 0-100 -> 0 to 12 samples mod depth, rate stays at 0.7 Hz
      - mix: store but do not apply (engine handles mix externally)
      - pre_delay: 0-100 -> 0 to 4800 samples (0-100ms)
    - Apply freeze if switch1 == -1
    - If switch1 == 1 (Bright): set damping to 0 regardless of knob
    - Call FDNCore.set_decay/set_damping/set_modulation/set_size/set_freeze as appropriate

    **reset(self):**
    - Reset FDNCore, all input diffusers, pre-delay line

    **param_specs property:**
    Return dict with exactly these keys and specs:
    ```
    "decay_time": type=knob, min=0, max=100, default=50, label="Decay Time"
    "size": type=knob, min=0, max=100, default=50, label="Size"
    "damping": type=knob, min=0, max=100, default=40, label="Damping"
    "modulation": type=knob, min=0, max=100, default=30, label="Modulation"
    "mix": type=knob, min=0, max=100, default=75, label="Mix"
    "pre_delay": type=knob, min=0, max=100, default=0, label="Pre-Delay"
    "switch1": type=switch, positions=[-1,0,1], default=0, label="Freeze", labels=["Freeze","Normal","Bright"]
    "switch2": type=switch, positions=[-1,0,1], default=0, label="Stereo Mode", labels=["Mono","Stereo","Wide"]
    ```

    **to_c_struct(self) -> str:**
    - Return a string containing a C typedef struct for the FDN state
    - Include: 4 delay buffers (float arrays with sizes), 4 write indices, 4 damping filter states (y1), 4 DC blocker states (x1, y1), LFO phase, gains[4], delay_lengths[4], mod_depth, lfo_inc, input diffuser states (3 allpass buffers + indices + gains), pre_delay buffer + index
    - Follow the pattern established in freeverb.py

    **to_c_process_fn(self) -> str:**
    - Return a string containing a C void function signature and body outline
    - Include comments explaining the signal flow: pre-delay -> diffusion -> FDN core -> stereo tap -> width -> clip
    - Does not need to be compilable -- just a template showing the structure
    - Follow the pattern established in freeverb.py

    **Important:** When implementing the processing loop, remember that the engine sets mix=100 internally and handles wet/dry blending itself. The mix knob value should be stored but the FDN should output 100% wet signal.
