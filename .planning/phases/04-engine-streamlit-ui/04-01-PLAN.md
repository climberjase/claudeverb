  <action>
    First add `streamlit>=1.38` to pyproject.toml dependencies list.

    Then write tests/test_engine.py with all behavior tests above. Use the bundled "impulse" sample from audio.samples.get_sample("impulse") as test audio (short, fast to process). Set matplotlib backend to "Agg" at top of test file.

    Then implement claudeverb/engine.py with three public functions:

    1. `process_audio(algorithm_name: str, params: dict, audio: np.ndarray) -> dict`:
       - Looks up algorithm class from ALGORITHM_REGISTRY[algorithm_name]
       - Instantiates algorithm, forces mix=100 in params (overrides any user-supplied mix value)
       - Calls update_params then reset, then process(audio.copy())
       - Generates IR via generate_impulse_response(algo, duration_seconds=3.0, mono=True) -- must call update_params again and reset before IR generation since process() consumed state
       - Computes all metrics (RT60 broadband, RT60 bands, DRR, C80, C50, spectral centroid delta) and formats as strings
       - Generates mel and fft comparison figures
       - Returns dict with keys: dry, wet, ir, metrics (dict of str->str), figures (dict of str->Figure)

    2. `blend_wet_dry(dry: np.ndarray, wet: np.ndarray, mix_pct: int) -> np.ndarray`:
       - Computes (1 - alpha) * dry + alpha * wet where alpha = mix_pct / 100.0
       - Hard clips result to [-1.0, 1.0]
       - Returns float32

    3. `plot_waveform_comparison(dry: np.ndarray, wet: np.ndarray, sr: int = SAMPLE_RATE) -> plt.Figure`:
       - Mono-izes stereo inputs (average channels)
       - Creates 2-row subplot: top=input waveform, bottom=output waveform
       - Time axis in seconds, amplitude axis [-1, 1]
       - Returns matplotlib Figure (caller closes)

    Run tests with `pytest tests/test_engine.py -x`.
