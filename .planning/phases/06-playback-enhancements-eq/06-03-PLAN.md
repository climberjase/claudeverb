  <action>
    Modify `claudeverb/streamlit_app.py` to add all Phase 6 UI features. This is a single-file change but touches multiple sections of the sidebar and main area.

    **Sidebar changes (in order of appearance):**

    1. **Multi-file sample selector** (replace current audio source section):
       - Replace `list_samples()` import with `list_all_samples` from `claudeverb.audio.samples`
       - Use `st.selectbox("Sample", list_all_samples())` for bundled sample selection
       - Keep "Upload File" radio option working as before
       - Session state key: `selected_sample`

    2. **Silence padding slider** (after audio source, before algorithm):
       - `st.slider("Silence Padding (seconds)", 0.0, 5.0, 2.0, step=0.1, key="silence_padding")`
       - Place after sample selection, near audio source (per user context -- it modifies input)

    3. **Loop toggle** (near audio source):
       - `st.checkbox("Loop Playback", value=False, key="loop_playback")`
       - Used later when calling `st.audio(..., loop=loop_value)`

    4. **Dattorro preset dropdown** (after algorithm selector, conditionally visible):
       - Only show when `algo_name == "dattorro_plate"`
       - Import `list_presets, get_preset, DATTORRO_PRESETS` from `claudeverb.algorithms.dattorro_presets`
       - Add session state key `preset_name`, default "Custom"
       - `st.selectbox("Preset", ["Custom"] + list_presets(), key="preset_name")`
       - When preset is selected (not "Custom"):
         - Write all preset param values to session state keys (`knob_decay`, `knob_bandwidth`, etc., and `switch_switch1`, `switch_switch2`)
         - Store preset values in `st.session_state["_preset_values"]` for comparison
         - Call `st.rerun()` to refresh knob positions
       - After knob rendering: compare current widget values to stored `_preset_values`. If any differ, set `preset_name` to "Custom" (but do NOT rerun -- just update the label display)

    5. **EQ controls** (after Wet/Dry slider, before Process button):
       - `st.checkbox("Enable EQ", value=False, key="eq_enabled")`
       - If enabled, show inline sliders:
         - Low Shelf: frequency `st.slider("Low Freq", 80, 500, 200, key="eq_low_freq")`, gain `st.slider("Low Gain (dB)", -12.0, 12.0, 0.0, step=0.5, key="eq_low_gain")`
         - Mid Parametric: frequency `st.slider("Mid Freq", 200, 5000, 1000, key="eq_mid_freq")`, gain `st.slider("Mid Gain (dB)", -12.0, 12.0, 0.0, step=0.5, key="eq_mid_gain")`, Q `st.slider("Mid Q", 0.5, 5.0, 1.5, step=0.1, key="eq_mid_q")`
         - High Shelf: frequency `st.slider("High Freq", 2000, 16000, 6000, key="eq_high_freq")`, gain `st.slider("High Gain (dB)", -12.0, 12.0, 0.0, step=0.5, key="eq_high_gain")`

    **Process handler changes:**

    - Collect `silence_seconds` from session state
    - Build `eq_params` dict from session state if EQ is enabled:
      ```python
      eq_params = None
      if st.session_state.get("eq_enabled"):
          eq_params = {
              "enabled": True,
              "low_freq": st.session_state.get("eq_low_freq", 200),
              "low_gain": st.session_state.get("eq_low_gain", 0.0),
              "mid_freq": st.session_state.get("eq_mid_freq", 1000),
              "mid_gain": st.session_state.get("eq_mid_gain", 0.0),
              "mid_q": st.session_state.get("eq_mid_q", 1.5),
              "high_freq": st.session_state.get("eq_high_freq", 6000),
              "high_gain": st.session_state.get("eq_high_gain", 0.0),
          }
      ```
    - Call `engine.process_audio(algo_name, params, audio_data, silence_seconds=silence_seconds, eq_params=eq_params)`
    - Store `original_length` from results in session state for waveform padding marker

    **Audio player changes:**

    - Get `loop_value = st.session_state.get("loop_playback", False)`
    - Pass `loop=loop_value` to all `st.audio()` calls

    **Waveform display changes:**

    - After calling `engine.plot_waveform_comparison(dry, wet)`, add padding boundary visualization:
      - If `original_length` is stored in results and less than total length:
        - Calculate boundary time: `boundary_s = original_length / SAMPLE_RATE`
        - Add `axvspan(boundary_s, total_length / SAMPLE_RATE, alpha=0.15, color='gray', label='Padding')` to both axes of the waveform figure
        - Alternative: modify `plot_waveform_comparison()` in engine.py to accept optional `padding_start_sample` parameter. If provided, shade from that sample to the end with light gray fill.

    **Session state initialization:**
    Add new keys to the initialization block at the top:
    ```python
    for key in ["dry_audio", "wet_audio", "results", "audio_loaded", "original_length"]:
        if key not in st.session_state:
            st.session_state[key] = None
    ```

    **IMPORTANT pitfall handling (from RESEARCH.md):**
    - Preset dropdown + knob snapping: When preset selected, write param values to session state THEN st.rerun(). Same pattern as existing "Reset to Defaults" button.
    - "Custom" detection: Compare current widget values vs stored `_preset_values` dict. If any differ, display "Custom" label. Do NOT update the selectbox value programmatically (causes infinite rerun loop).
    - Silence padding length: dry and wet are both padded (engine handles this), so blend_wet_dry works without shape mismatch.
