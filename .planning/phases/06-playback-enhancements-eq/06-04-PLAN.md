## Root Cause (from UAT diagnosis)

When the user switches away from Dattorro Plate, the cleanup block at line 101
does:

```python
for k in ["results", "dry_audio", "wet_audio", "preset_name", "_preset_values"]:
    st.session_state[k] = None
```

Setting `preset_name` to `None` in session_state means Streamlit's `selectbox`
(which has `key="preset_name"`) picks up `None` as its stored value on the next
run. The `index` argument is ignored when a key already exists in session_state.
So `preset_name` evaluates to `None`.

The guard at line 124 is:

```python
if preset_name != "Custom":
    preset_vals = get_preset(preset_name)   # <-- crashes: get_preset(None)
```

`None != "Custom"` is `True`, so `get_preset(None)` is called and raises `KeyError`
because `None` is not a key in `DATTORRO_PRESETS`.

## Two-point fix

1. In the cleanup loop, **pop** the `preset_name` key rather than setting it to
   `None`. When the key is absent, the `selectbox` falls back to its `index=0`
   default (i.e., "Custom").

2. Strengthen the guard from `preset_name != "Custom"` to
   `preset_name and preset_name != "Custom"` so any residual `None` cannot reach
   `get_preset()`.
