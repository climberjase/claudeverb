Two root causes to fix:

**Audio playback quality (Gap 2):**
The `st.audio()` calls pass numpy arrays in claudeverb's (2, N) channels-first format, but Streamlit expects (N, channels) channels-last for stereo. This causes garbled/distorted playback for stereo audio.

Add a helper function `_prepare_for_playback(audio)` that:
- If audio is mono (N,), return as-is
- If audio is stereo (2, N), transpose to (N, 2) for Streamlit
- Ensure float32 dtype

Apply this helper to ALL three `st.audio()` calls:
1. Line 177: `st.audio(_prepare_for_playback(dry), sample_rate=SAMPLE_RATE, loop=True)`
2. Line 182: `st.audio(_prepare_for_playback(blended), sample_rate=SAMPLE_RATE, loop=True)`
3. Line 245: IR is already mono, but wrap for safety

**UI session state bugs (Gap 1):**
When switching algorithms, stale widget keys from the previous algorithm remain in session state, causing Streamlit DuplicateWidgetID errors or incorrect parameter values.

Fix by:
1. Track the current algorithm name in session state as "current_algo"
2. After the algorithm selectbox, check if algo changed: `if st.session_state.get("current_algo") != algo_name`
3. If changed: delete all "knob_*" and "switch_*" keys from session state, clear "results", "dry_audio", "wet_audio", set "current_algo" = algo_name, then call `st.rerun()`
4. Also store matplotlib figures outside session state or close them immediately after rendering to prevent memory accumulation. The current code already calls plt.close() after st.pyplot(), which is correct -- verify this is consistent for all figures.

**Additional hardening:**
- Add `st.session_state.setdefault("current_algo", algo_name)` in the initialization block at the top
- In the reset handler, also reset "current_algo" so defaults apply cleanly
