  <action>
Modify `claudeverb/streamlit_app.py` to add two new UI sections:

**A. Signal-Flow Diagram (main content area):**
1. Add a new expander or tab alongside the existing waveform/spectrogram views labeled "Signal Flow"
2. Inside, add a `st.radio` or `st.toggle` for detail level: "Block" (default) vs "Component"
3. Call `algo_instance.to_dot(detail_level, current_params)` where current_params is the dict of current knob values from the sidebar
4. Render with `st.graphviz_chart(dot_string)` -- do NOT use `use_container_width=True` (known Streamlit regression per research). Use default width or `width="stretch"` if available.
5. The diagram regenerates on every Streamlit rerun (which happens when any widget changes) -- no caching needed since to_dot() is cheap string formatting

**B. C Export (sidebar, below algorithm controls):**
1. Add a section header "C Export" in the sidebar
2. Add "Export to C" button (`st.button`)
3. When clicked, show an expander with:

   a. **Knob Mapping**: 6 `st.selectbox` dropdowns, one for each Hothouse knob (KNOB_1 through KNOB_6). Options are the algorithm's param_specs keys (knobs only, not switches). Default order matches param_specs iteration order. Store mapping in `st.session_state['knob_mapping']`.

   b. **Switch Mapping**: 2 `st.selectbox` for TOGGLESWITCH_1 and TOGGLESWITCH_2 mapping to the algorithm's switch params. TOGGLESWITCH_3 reserved for bypass.

   c. **RAM Estimation**: Call `estimate_ram(algo_instance)` and display the breakdown:
      - Per-component: "Delay lines: X.X KB, Filters: X.X KB, Total: X.X KB"
      - Fit indicator: green checkmark if fits_sram, yellow warning if needs SDRAM, red X if too large
      - List SDRAM candidates if any

   d. **Code Preview**: Two `st.code` blocks (language="c") showing generated .h and .c content from `generate_header()` and `generate_source()`. Also show AudioCallback in a third code block from `generate_audio_callback()`.

   e. **Save Button**: `st.button("Save to daisyexport/")` that calls `export_to_files()` and shows `st.success()` with file paths written.

**Session state management:**
- `st.session_state['export_preview']`: bool, whether export panel is expanded
- `st.session_state['knob_mapping']`: dict of knob assignments
- Reset knob_mapping when algorithm changes (detect via comparing current algo name to `st.session_state.get('export_algo')`)

**Important:** Do NOT use `use_container_width=True` on `st.graphviz_chart()` -- use default sizing or `width="stretch"`.
