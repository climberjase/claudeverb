  <name>Task 1: Audio I/O module with load/save</name>
  <files>claudeverb/audio/__init__.py, claudeverb/audio/io.py, tests/test_audio_io.py</files>
  <behavior>
    - load() returns float32 ndarray at 48 kHz for any supported format (WAV, AIFF, FLAC, OGG)
    - load() returns shape (N,) for mono, (2, N) for stereo
    - load() handles >2 channel files by taking first two channels
    - load() always returns float32 (never float64, even from librosa edge cases)
    - save() writes WAV file readable by soundfile, default 32-bit float subtype
    - save() transposes (2, N) stereo to (N, 2) for soundfile convention
    - save() with subtype="PCM_24" produces 24-bit PCM output
    - Round-trip: load(save(audio)) preserves data within float32 precision
  </behavior>
  <action>
    Create `claudeverb/audio/__init__.py` re-exporting load, save from io.py.
    Create `claudeverb/audio/io.py` with:
    - `load(path: str) -> np.ndarray`: Uses `librosa.load(path, sr=SAMPLE_RATE, mono=False)`. Always cast result to float32. Handle mono (ndim==1 -> return as-is), stereo (shape[0]==2 -> return as-is), multichannel (shape[0]>2 -> take first 2 channels, log warning). Log warning for files >60 seconds.
    - `save(path: str, audio: np.ndarray, subtype: str = "FLOAT") -> None`: Uses `soundfile.write()`. Transpose stereo (2, N) to (N, 2) before writing. Write at SAMPLE_RATE.

    Create `tests/test_audio_io.py` with tests:
    - test_load_mono_wav: Generate a short mono WAV via soundfile, load it, verify shape (N,), dtype float32
    - test_load_stereo_wav: Generate stereo WAV, verify shape (2, N), dtype float32
    - test_load_resamples_to_48k: Generate WAV at 22050 Hz, load, verify length matches 48kHz duration
    - test_load_multichannel_takes_first_two: Generate 4-channel WAV, verify shape (2, N)
    - test_save_and_reload_roundtrip: Save float32 array, reload, verify data matches within tolerance
    - test_save_stereo_transposes: Save (2, N) array, verify file is readable and correct
    - test_load_returns_float32_not_float64: Verify dtype is always np.float32

    Use tmp_path pytest fixture for all file operations. Use soundfile directly in tests to create input WAV files (not through our load/save).
  </action>
